# Reach — SEO / landing-page honesty & polish

**Status:** approved design · **Date:** 2026-06-27 · **Type:** pipeline (distill.py) SEO fixes
**Branch:** `feature/reach-seo` (off `main` — ①②③ already merged)

## Overview

Workstream ④ (final) of the post-audit program. Three focused, honesty-leaning fixes to the SEO surface,
all in `scripts/distill.py` (the `/b` per-ascendancy landing pages + `sitemap.xml`), locally testable on
Python 3.14.6 and regenerated by the hourly `distill.yml` / on the next data refresh:

- **A. Sitemap lists `data.json` as a page.** `write_sitemap` (distill.py:476) emits a `<url>` for
  `…/data.json` — a JSON data file is not a crawlable page; crawlers shouldn't be told to index it. Drop it.
- **B. `/b` pages have no Twitter Card meta.** The SPA (`index.html:30-33`) has `twitter:card` etc., but
  `landing_html` emits only `og:` tags — so a shared `/b` link gets no rich Twitter/X preview. Add the four
  `twitter:*` tags (consistent with the SPA + the existing `og:` block).
- **C. The `/b` weapon claim is dishonest on live data.** `landing_html` asserts *"Typically a {weapon}
  build."* from `meta-detail.json`'s dominant `weaponmode`, but the live data exposes two failures, baked
  into indexed `<meta description>`s: (1) **"Unknown" leaks in** — Stormweaver → "Typically a **Unknown /
  Sceptre** build", Shaman → "**Dual Unknown**"; (2) it says **"Typically"** even at a weak **15–19%**
  plurality (Titan "Staff 15%", Infernalist 16%, Acolyte/Ritualist 18%). At 15% "typically" is false.

### Out of scope (verified non-issues / by-design)

- **Fonts** — already fully self-hosted (`index.html:8/38/40`, `fonts.css → fonts/*.woff2`, "NO third-party").
  The "Google Fonts" wording lived only in the gitignored `CLAUDE.md`; nothing to fix on the site.
- **`/b` "no live share"** — by design: the `/b` pages are stable-content (no hourly churn) and explicitly
  defer to the SPA for live share ("See the live {asc} meta … on Tincture"). Honest as-is.
- **Node 20 GitHub-Actions deprecation warning** — CI infra, GitHub auto-forces Node 24; a trivial action
  version bump, deferred as a separate chore (not user-facing Reach).

## Goals

- No data file is advertised as a crawlable page in the sitemap.
- A shared `/b` link renders a rich Twitter/X card (matching the SPA).
- The `/b` weapon claim is **honest**: shown only when there's a genuinely dominant weapon (≥ 30% share) whose
  name carries no "Unknown", and phrased factually ("Most-played weapon: {weapon}.") — never "typically" at a
  weak plurality, never "Unknown" in an indexed description.

## Non-goals (YAGNI)

- No change to the `/b` stable-content model (no live share on `/b`), the SPA, the `og:` image, the ranking,
  or the data pipeline's share/trend math.
- No new landing-page content sections; no change to ascendancies that legitimately have no dominant weapon
  (they simply omit the weapon line — honest).

## Honesty guardrails

- A weapon is named only when the data supports "this is the build's weapon" — a clear dominant share, no
  unclassified ("Unknown") main-hand. Below the threshold, omit it (silence is honest; a 15% plurality is not
  "the weapon").
- "Most-played weapon: {weapon}." is a plurality fact, not a majority/typicality claim.

In `landing_html`'s head `parts` (after the `og:image:height` line, distill.py:719), add:

## C — `/b` weapon label: gate on a dominant, clean weapon + factual wording

1. **Gating helper** (pure, testable) in `distill.py`:
   ```python
   WEAPON_DOMINANT_PCT = 30   # below this there's no single "build weapon" worth claiming

   def _dominant_weapon(entry):
       """The ascendancy's weapon to name on its /b page — only a clearly dominant, classified one.
       None when the plurality is weak (<WEAPON_DOMINANT_PCT) or the weaponmode is unclassified ('Unknown')."""
       w = (entry.get("weapons") or [{}])[0]
       name = w.get("name")
       if not name or "Unknown" in name or (w.get("pct") or 0) < WEAPON_DOMINANT_PCT:
           return None
       return name
   ```
2. **Wire into `write_landing`** (distill.py:786): replace
   `weapon = (e.get("weapons") or [{}])[0].get("name") if e.get("weapons") else None`
   with `weapon = _dominant_weapon(e)`.
3. **Factual wording** in `landing_html` (distill.py:680): replace
   `wbit = f" Typically a {weapon} build." if weapon else ""`
   with `wbit = f" Most-played weapon: {weapon}." if weapon else ""`.
   (The body weapon chip at line 731 stays — it already only renders when `weapon` is truthy, so the gating
   hides it automatically for weak/Unknown cases.)

Net: ~16 ascendancies keep a clean weapon claim ("Most-played weapon: Bow / Quiver."), the 5 weak-plurality
ones (Titan/Infernalist/Acolyte/Ritualist/Gemling) and the 2 "Unknown" ones (Stormweaver/Shaman) omit it.
