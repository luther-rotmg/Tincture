**Goal:** Make the SEO surface honest and complete — drop `data.json` from the sitemap, give the `/b` landing pages Twitter cards, and stop the `/b` pages from claiming "Typically a {weapon} build" when the weapon is a weak plurality or "Unknown" — then regenerate + commit the real pages so the fix is live on merge.

## Global Constraints

- **Honesty:** no `/b` page may say "Typically a {weapon} build" — a weapon is named only when its share ≥ **30%** (`WEAPON_DOMINANT_PCT`) AND its name contains no **"Unknown"**; the wording is **"Most-played weapon: {weapon}."** No "Unknown" or "Typically" in any indexed `<meta>`/og/ld+json text.
- **Scope:** `scripts/distill.py`, `scripts/test_distill.py`, and the regenerated artifacts `b/*.html`, `b/index.json`, `sitemap.xml`. No SPA (`index.html`) change, no data-pipeline math change, no `--demo` (it overwrites `data.json` with sample data).
- **Refactor fidelity:** the extracted `_sitemap_xml` must produce byte-identical output to today's `write_sitemap` **minus** the `data.json` `<url>` (same 2-space indent, homepage-then-sorted-`/b` ordering, `</urlset>\n` trailing newline).
- **Stdlib only**, fail-safe. Commit only the files each task names (`git add <files>`) — the tree may carry a modified `economy.json`; never stage it.
- **Branch:** `feature/reach-seo` (off `main`). Commit after every task. **Line numbers shift** as tasks add helpers above existing functions — match by the quoted anchor text, not the number.

def _sitemap_xml(lastmod, asc_slugs=None):
    """The sitemap XML string: the homepage + each per-ascendancy /b landing page. A JSON data file
    (data.json) is NOT a crawlable page, so it is deliberately excluded. Pure — no IO."""
    def url_block(loc, prio, freq="hourly"):
        return (f"  <url>\n    <loc>{loc}</loc>\n    <lastmod>{lastmod}</lastmod>\n"
                f"    <changefreq>{freq}</changefreq>\n    <priority>{prio}</priority>\n  </url>\n")
    blocks = url_block(SITE_URL, "1.0")
    for slug in sorted(asc_slugs or []):
        blocks += url_block(f"{SITE_URL}b/{slug}.html", "0.6", "weekly")
    return ('<?xml version="1.0" encoding="UTF-8"?>\n'
            '<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">\n'
            + blocks + "</urlset>\n")
```
(`write_sitemap` keeps its docstring, the `lastmod = …` line above the call, and the `try/except` file write below it — only the XML-building block moves into the helper, minus the `data.json` block.)

(3a) Add **module-level**, immediately above `def landing_html(...)`:

WEAPON_DOMINANT_PCT = 30   # below this share there's no single "build weapon" worth naming on a /b page


def _dominant_weapon(entry):
    """The ascendancy's weapon to name on its /b page — only a clearly dominant, classified one.
    None when the plurality is weak (< WEAPON_DOMINANT_PCT) or the weaponmode is unclassified ('Unknown')."""
    w = (entry.get("weapons") or [{}])[0]
    name = w.get("name")
    if not name or "Unknown" in name or (w.get("pct") or 0) < WEAPON_DOMINANT_PCT:
        return None
    return name
```
